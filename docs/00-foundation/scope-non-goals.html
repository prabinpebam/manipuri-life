Scope & Non-Goals

What is in v1, what is explicitly out, and where the 2% convenience budget applies - but first and foremost, the framework we use to decide any of it.

TL;DR

Scope stays honest with one rule: every candidate feature must serve a pillar and earn its place in the vertical slice, or it is cut or deferred. This page holds the decision framework - a pillar + MoSCoW rubric and the 98/2 budget - plus a draft in/out list to react to. Nothing is locked; the framework matters more than today's list.

Draft for review The framework is the point

The specific in/out list below will change many times. What should stabilise first is how we decide - so that every future "should this be in the game?" has a repeatable answer instead of a debate.

The decision framework

Every proposed feature, system, or piece of content runs this gauntlet. If it survives, it has a place; where it lands (now, later, never) falls out of the scoring.

  1. 1 · Which pillar does it serve?

    Name the pillar it advances - a great game first, authenticity is the soul, progress you can see, calm is a feature, understanding by living, or specific not generic. If you can't name one, it's out.

  2. 2 · Vertical-slice test

    Is it needed to prove the core loop in the first playable slice? If yes, it's a candidate for v1 core. If no, it defers - however good it is.

  3. 3 · MoSCoW placement

    Sort it into Must / Should / Could / Won't (v1). "Must" is reserved for things the game is broken without.

  4. 4 · The 98/2 gate

    If it bends history, it spends from the 2% playability budget - and the deviation must be written down, never silent (see How We Work).

  5. 5 · Cost vs payoff

    Weigh effort and risk against player value. Prefer high-payoff, low-risk; be suspicious of high-effort features that serve only one pillar weakly.

The test that keeps scope honest

A feature earns its place only if you can finish this sentence: "We are building this because it serves <pillar>, and the vertical slice needs it because <reason>." If the sentence stalls, the feature waits or dies. Scope creep is almost always a feature that can't finish that sentence.

Non-goals - what we refuse to build

Saying no clearly is how the yes stays meaningful. These are out on purpose, and why.

swords

A combat / action game

Twitch challenge and violence fight "calm is a feature." Any conflict is gentle, optional narrative weather - never a stress mechanic.

groups

Multiplayer / live-service

Single-player, offline-first. No servers, no retention treadmills - they add cost and pull against a calm, private experience.

casino

A procedural / generic sandbox

A hand-authored, specific Manipur - not a random village generator. "Specific, never generic" rules this out.

auto_fix_high

Fantasy with magic as real

Folklore and belief appear exactly as people believed them; no proven magic or monsters. The 98/2 rule forbids it.

The 2% convenience budget

Scope has a small, deliberate escape hatch. 98% historical reality, 2% gameplay convenience - spent on purpose, documented, and never used to smuggle in inaccuracy for its own sake.

Fair use of the 2%

  • Gentle time compression so a day is a satisfying session
  • Quality-of-life UI that never contradicts the record
  • Small readability liberties (legible signage, forgiving hitboxes)

Not what it's for

  • Anachronistic crops, tools, or customs "because they're fun"
  • Invented festivals or borrowed traditions
  • Silent deviations - every spend is written down

Open questions before we lock scope

  • Romance in v1? A "Could" today - does relationship depth need it to land, or is friendship enough?
  • One village or several? Map size trades authenticity depth against production cost.
  • How much economy? Enough to feel a livelihood, without becoming an optimisation treadmill.
  • What is the minimum vertical slice? The smallest thing that proves the whole game is worth building.
