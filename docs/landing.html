Manipur 1700s — Documentation Hub

The single source of truth for a historically-accurate life-simulation game set in 18th-century Manipur. This site maps every document we must author to design, research, and build the project — and tracks how much of that debt is still open.

TL;DR

We are building a Stardew-Valley-style life sim set in the Meitei kingdom of the 1700s (the era of Garib Niwaz). The ambition is a playable ethnography: historical accuracy is a design constraint, not a coat of paint. Documentation is organised into nine domains, each detailed out slowly, page by page. Start with How We Work, then watch progress on the Documentation Backlog.

How the documentation is structured

Documentation flows top-down: a Foundation fixes vision and scope; a Historical Research layer (the "Knowledge Bible") feeds facts into every craft discipline; two cross-cutting domains — Production and Cultural Integrity — govern how the whole thing gets made.

Working principle

Don't build "Fantasy Manipur"

The biggest risk is a world that looks Manipuri but behaves like a generic farming sim. Every object, meal, tool, custom, and conversation should answer a question the player didn't know they had. When in doubt, the Knowledge Bible decides.
